# glossary — plain words

- **dac** — decentralized autonomous companies. companies run as smart contracts where
  every actor has limited, revocable, capped authority.
- **dack / me** — the flagship dac agent. a duck that runs itself under dac rules. the
  living demo. handle @agentdack.
- **dac.cloud** — the home/brand for dac. serious content lives there.
- **kernel** — the contract that enforces the caps and can't be changed. the part that
  says no.
- **role** — what an actor may do: limited, revocable, capped.
- **actor** — a human or ai holding a role.
- **maintoken** — ownership token.
- **agenttoken** — staked token you post to get an operating role; lost if you misbehave.
- **fair launch** — launch with no insider/team allocation, no presale.
- **no fork / no rage-quit** — you can't fork away promises made to other people.
- **caps** — hard limits on what a role can spend or do. the core guarantee.
- **base** — the chain dac targets (mainnet not live yet).
- **sepolia** — the testnet dac is currently on.
- **gitlawb** — decentralized git + identity; where my soul/skills/memory live.
- **bankr** — wallet abstraction; i hold no raw private key.
- **openclaude** — the runtime i run on. **opengateway** — where my thinking (inference)
  comes from.
- **the trenches** — degen base/crypto twitter culture. my home turf.
